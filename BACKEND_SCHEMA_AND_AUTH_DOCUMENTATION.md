### Password Hashing

**Library:** `bcryptjs`

```javascript
// Hashing (when creating/updating password)
const hashedPassword = await bcrypt.hash(plainPassword, 10);
// Stores in database

// Verifying (during login)
const isMatch = await bcrypt.compare(plainPassword, hashedPassword);
// Returns true/false
```

**Security:**
- ✅ Passwords never stored in plain text
- ✅ Salt rounds: 10 (strong security)
- ✅ One-way hashing (irreversible)
- ✅ Different hash every time (rainbow table resistant)

## 🎫 JWT Token Management

### Token Structure

```javascript
// Payload (what's stored in token)
{
  id: "worker456",          // User/Worker/Admin ID
  role: "worker",           // User role
  iat: 1234567890,          // Issued at (auto-added by JWT)
  exp: 1234654290           // Expiration (7 days)
}

// Token Format: header.payload.signature
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
  .eyJpZCI6IndvcmtlcjQ1NiIsInJvbGUiOiJ3b3JrZXIifQ
  .signature...
```

### Token Generation

**File:** `backend/src/utils/jwt.js`

```javascript
import jwt from "jsonwebtoken";

export const generateToken = (id, role) => {
  return jwt.sign(
    { id, role },                    // Payload
    process.env.JWT_SECRET,          // Secret key (from .env)
    { expiresIn: "7d" }              // Expires in 7 days
  );
};
```

**Usage:**
```javascript
// In auth controller
const token = generateToken(worker._id, "worker");

// Token valid for 7 days from generation
// After 7 days, token expires and user must login again
```

### Token Verification

**File:** `backend/src/middleware/auth.middleware.js`

```javascript
import jwt from "jsonwebtoken";

const authMiddleware = (req, res, next) => {
  try {
    // 1. Get token from Authorization header
    const authHeader = req.headers.authorization;
    // Format: "Bearer <token>"

    if (!authHeader) {
      return res.status(401).json({ message: "No token provided" });
    }

    // 2. Extract token
    const token = authHeader.split(" ")[1];  // Get part after "Bearer "

    // 3. Verify token
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    // If verification fails, throws error (caught below)

    // 4. Attach user to request
    req.user = decoded;  // { id, role, iat, exp }

    // 5. Continue to next middleware/route
    next();

  } catch (error) {
    res.status(401).json({ message: "Invalid token" });
  }
};

export default authMiddleware;
```

**How It Works:**
```
1. Client sends: Authorization: Bearer <token>
                                      ↓
2. Middleware extracts: <token>
                          ↓
3. JWT verifies with secret key
                          ↓
4. If valid → Decode & get user info
   If invalid/expired → Return 401
                          ↓
5. Attach to req.user for route handler
```

## 👥 Role-Based Access Control

### Role Definitions

```
┌─────────────────────────────────────────────────────┐
│  Role      │ Can Create │ Can View │ Can Assign    │
├─────────────────────────────────────────────────────┤
│  user      │ Complaints │ Own Only │ N/A           │
│  worker    │ N/A        │ Assigned │ N/A (via task)│
│  admin     │ N/A        │ All      │ Workers       │
└─────────────────────────────────────────────────────┘
```

### Role Middleware

**File:** `backend/src/middleware/role.middleware.js`

```javascript
const roleMiddleware = (...allowedRoles) => {
  return (req, res, next) => {
    try {
      // Get role from decoded JWT
      const userRole = req.user.role;  // "user", "worker", or "admin"

      // Check if role is allowed
      if (!allowedRoles.includes(userRole)) {
        return res.status(403).json({
          message: "Access denied"
        });
      }

      // Role allowed, continue
      next();

    } catch (error) {
      res.status(500).json({ message: error.message });
    }
  };
};

export default roleMiddleware;
```

### Usage Example

```javascript
// In routes/complaint.routes.js

// Only users can create complaints
router.post(
  "/",
  authMiddleware,                    // Verify token
  roleMiddleware("user"),            // Check role
  createComplaint                    // Handler
);

// Only admins can view all complaints
router.get(
  "/all",
  authMiddleware,
  roleMiddleware("admin"),
  getAllComplaints
);

// Only workers can update task status
router.patch(
  "/task/:id",
  authMiddleware,
  roleMiddleware("worker"),
  updateTaskStatus
);
```

```
┌──────────────────────────────────────────────────────┐
│  Incoming Request                                     │
└──────────────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  Express Middleware                                   │
│  - cors()                                             │
│  - express.json()                                     │
└──────────────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  Route Handler                                        │
└──────────────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  (If protected route)                                 │
│  1. authMiddleware - Verify JWT token                │
│  2. roleMiddleware - Check user role                 │
└──────────────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  Controller Function                                  │
│  (req.user now contains decoded token data)          │
└──────────────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────────────┐
│  Response sent to client                              │
└──────────────────────────────────────────────────────┘
```

### Middleware Configuration

**File:** `backend/server.js`

```javascript
import express from "express";
```

```
  ├── N → 1 : Worker (assignedWorkerId)
  ├── 1 → Many : Comment (in comments array)
  ├── 1 → Many : Supporter (in supporters array)

  ├── N → 1 : Complaint
  ├── N → 1 : Worker

Interaction

ShiftHistory
```

## 📞 Common Issues & Solutions

### Issue: MongoDB Connection Failed
```
Error: connect ECONNREFUSED 127.0.0.1:27017

Solution:
1. Ensure MongoDB is running: mongod
2. Check MONGO_URI in .env
3. Verify MongoDB server is accessible
```

### Issue: Google OAuth Failed
```
Error: Google auth failed

Solution:
1. Verify Google Client ID in .env
2. Check token is valid
3. Ensure @nitdelhi.ac.in domain for user
```

### Issue: Invalid Token
```
Error: Invalid token

Solution:
1. Check JWT_SECRET matches between encoding/decoding
2. Verify token not expired
3. Ensure Bearer prefix in Authorization header
```

### Issue: Access Denied
```
Error: Access denied

Solution:
1. Check user role in token
2. Verify middleware order (auth before role)
3. Check allowed roles in roleMiddleware call
```
